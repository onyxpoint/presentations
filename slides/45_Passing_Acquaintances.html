  • Similar to Puppet in terms of focus
  • Much more developer focused, no DSL
  • Operations are strictly ordered and failures are immediate
    • Ruby rescue statements can be used to bypass failure states
  • Cookbooks are pulled onto the various nodes and executed independently

  • One of the oldest configuration management systems
  • Written in C for speed
  • Uses a type of DSL that feels similar to C
  • Unlike Puppet or Chef, requires more low-level programming understanding to modify the core
