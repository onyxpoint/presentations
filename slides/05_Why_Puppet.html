Why Use Puppet

  • You have numerous systems that you need to keep under control
  • You may want to repeat your system configuration again exactly at some other time
    • This applies particularly for rapidly evolving OSs like Fedora
  • You need to manage many different flavors of OSs
    • Linux - Red Hat, Ubuntu, Gentoo, etc...
    • UNIX - Mac OS X, Solaris, AIX, HP-UX
    • Microsoft Windows (yeah, them too)

Ways to Use Puppet

  • Server/Client (puppet agent)
    • Traditional Model
    • Provides for Server Controlled System Isolation
    • PKI Communication Protection
    • Can also make all nodes a server!
  • Standalone (puppet apply)
    • Apply a manifest locally to the system
  • Precompiled Catalog (puppet apply)
    • Apply a compiled catalog locally to a system
